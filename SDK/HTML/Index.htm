FX.DLL 1.03 SDK
FX Technology

FX library was developed for programmers working within Visual Basic. The FX technology offers you fast, powerful and safe on Graphical Device Interface (GDI) based solutions.

FX library is written in Microsoft Visual C++ 6.0 and is working in Win32 subsystem. FX library supports all Windows platforms, except for the historical Windows 3x.

The FX technology can be powerfull solution to graphical problems if you prefer working within GDI. Learn, how to get the maximum out of the FX library with this SDK!

Frequently Asked Questions (FAQ)

How to use FX.DLL with Visual Basic?

FX.DLL 1.03 SDK comes not only with FX.DLL. There is also included BAS file (Visual Basic module) in package. FX.BAS is main key to access FX library easy and fast, as it contents all FX.DLL API declarations, constant and tag definitions.

To use FX.DLL in your Visual Basic projects simply copy the FX.BAS file to your project folder. Then open your Visual Basic project, on menu click Project, point to Add Module and click it. Wait until browse window opens. Then on tab choose Existing. Select the FX.BAS module and experience the FX technology!

 

Where the FX.DLL will render bit-maps?

As already you could imagine, FX.DLL uses Device Context (DC) as surface where bit-maps will be displayed. This means that you can use any object that has DC as surface for rendering.

I suggest using Picture Boxes, because they are easy to use. Always remember to set following Picture Box properties to be able to render bit-maps on their surface correctly:

AutoRedraw = True

ScaleMode = vbPixels

If you want to use Picture Box as invisible back-buffer, use following property values:

AutoRedraw = True

AutoSize = True

ScaleMode = vbPixels

Visible = False

Of course you can and always should use memory DCs as back-buffers instead of Picture Boxes, because memory DCs works much faster and saves some system resources as well.

 

How the rendering should be done?

When you are rendering bit-maps with FX.DLL you must always remember to clear DC at first place and update it at last place. This can be done very easy, if you use Picture Box. Simply call .Cls to clear the DC and .Refresh to update it. The simple example of rendering in Picture Box looks like this:

Private Sub Update()

    Picture1.Cls

    fxText Picture1.hDC, 10, 10, "Hello World!", _

        vbBlue, TA_LEFT

    Picture1.Refresh

End Sub
